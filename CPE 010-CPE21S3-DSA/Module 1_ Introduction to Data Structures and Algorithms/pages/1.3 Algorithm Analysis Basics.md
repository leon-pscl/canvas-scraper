Goal of Analysis of Algorithm
There are numerous ways to go from city "A" to city "B," including bicycle, bus, train, and air travel. We decide on the best option based on accessibility and convenience. We can choose the algorithm that uses the least amount of time and space by analyzing them.

The goal of algorithm analysis is to compare algorithms in terms of running time (primarily) and other factors (such as memory, developer effort and etc.).
What is running time analysis?
It involves figuring out how processing time grows as problem size (or input size) increases. The amount of elements in the input is known as the input size, and depending on the nature of the problem, the input may come in a variety of forms. The most typical sorts of inputs are listed below.

Size of an array

Polynomial degree

Number of elements in a matrix

Number of bits in the binary representation of the input

Vertices and edges in a graph.
